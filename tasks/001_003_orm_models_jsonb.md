# TASK 001_003_orm_models_jsonb
## Назначение: Реализация ORM-моделей таблиц (`users`, `user_filters`, `vacancies`, `match_logs`).
## Входные данные: Подключенный пул SQLAlchemy.
## Что сделать: 
1. В `app/core/models.py` определить классы моделей, маппингуя поля к колонкам PostgreSQL.
2. Для `user_filters` строго использовать тип данных `JSONB` для поля `weights_config`.
3. Настроить FK-связи с `on_delete='CASCADE'`, индексы на `(user_id, created_at)` и GIN-индексы для JSONB при необходимости.
## Ожидаемый результат: Файл `models.py` с 4 таблицами, готовыми к миграции в БД.
## Как протестировать: Скрипт `test_models.py`: импортирует модели, вызывает `create_all()` (или проверяет существование таблиц), создаёт запись с JSONB-полями и подтверждает тип данных через сырой SQL query.
