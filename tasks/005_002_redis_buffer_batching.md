# TASK 005_002_redis_buffer_batching
## Назначение: Буферизация рассылок (Notification Batching) через Redis с учётом `cooldown_interval`.
## Входные данные: Запущенный бот-дипатчер.
## Что сделать: 
1. Структура буфера в Redis: `KEY = user_id:cooldown_bucket` -> массив ID вакансий.
2. Логика таймера: сообщения накапливаются, отправляться пакетами через интервал `cooldown_interval`.
3. Группировка релевантных находок в один пост-канал ("Ваша подборка за период").
## Ожидаемый результат: Умная рассылка без спама, группирующая уведомления по таймеру юзера.
## Как протестировать: Скрипт эмулирует 3 релевантные вакансии для юзера, проверяет через Redis, что они попали в буфер, и после истечения `cooldown_interval` формирует 1 пакетное сообщение.
