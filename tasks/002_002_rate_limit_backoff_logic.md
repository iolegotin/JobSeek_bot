# TASK 002_002_rate_limit_backoff_logic
## Назначение: Обработка лимитов API hh.ru (429 ошибки) и экспоненциальная задержка.
## Входные данные: Базовый асинхронный клиент HH.ru.
## Что сделать: 
1. Добавить декоратор или мидлвар-обёртку для запросов, контролирующих счётчик в Redis (`HINCRBY`).
2. При получении `HTTP 429 Too Many Requests` реализовать логику Backoff: $T = T_{base} \cdot 2^n$, замораживание парсера и алерт админу.
3. Кэширование локально загруженных данных для повторной обработки при восстановлении сети.
## Ожидаемый результат: Механизм безопасного парсинга с защитой от бана API и кэшированием при сбоях.
## Как протестировать: Мокировать ответ сервера с `429`, запустить таймер, проверить через `time.sleep()` или асинхронный луп, что клиент корректно ставит паузу и отправляет алерт в TG-чат админа.
