# TASK 003_001_pyrogram_floodwait_handler
## Назначение: Настройка юзер-бота для парсинга каналов с защитой от `FloodWaitError`.
## Входные данные: Конфиг Pyrogram/Telethon, файл `.session` или API token.
## Что сделать: 
1. В `app/core/services/tg_fetcher.py` инициализировать клиент User-Bot.
2. Написать обработчик исключений `FloodWaitError`, который ставит случайную задержку (Jitter 2–5 сек) между запросами.
3. Реализовать инкрементальный захват: хранение `last_offset` или `max_id` для каждого целевого канала.
## Ожидаемый результат: Безопасный парсер каналов, не вызывающий банов и умеющий восстанавливаться после FloodWait.
## Как протестировать: Скрипт подключается к тест-каналу, имитирует искусственный `FloodWait`, проверяет корректную паузу (sleep) и получение сообщений после таймера.
