# TASK 003_002_incremental_deduplication
## Назначение: Склейка дубликатов вакансий из TG-каналов по хэшу текста поста.
## Входные данные: Работающий юзер-бот с логикой `max_id`.
## Что сделать: 
1. Вычисление `hash(text_post + channel_id)` перед сохранением в БД.
2. Проверка наличия хэша в Redis (`EXPIRE 7 days`). При совпадении вакансия игнорируется и не пишется в ORM.
3. Формирование инкрементальных пакетов данных для снижения нагрузки на сеть.
## Ожидаемый результат: Фильтр дубликатов, экономящий память БД и трафик.
## Как протестировать: Отправить 2 сообщения с идентичным текстом в тест-канал, проверить через Redis, что второе сообщение отброшено как дубль.
